Want to contribute to PLearn?

Keeping up-to-date

The latest code can always be checked out through anonymous (pserver) CVS with the following instructions. When prompted for a password for anonymous, simply press the Enter key.

cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/plearn login
cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/plearn co PLearn

Then, to keep up-to-date with the code, you just have to regularly run cvs update -d -P from within your checked-out PLearn directory.

I insist that this is only if you want to keep up with the rapid life-cycle of the library, in which case you should also subscribe to the developers mailing-list. If you want stability, stick with the official tarball releases.

Occasional contributions

If you have found a bug, or fixed one, or written a piece of code that you think might be worth including in the PLearn library, just send me an email about it at vincentp@iro.umontreal.ca

Setting up CVS with write access

If you are to become a regular contributor, trusted with direct write permission to the CVS repository, you must check out PLearn in a different way.

Follow the steps described below:
  1. If you don't already have one, open an account on sourceforge.net: go there, and try your luck for a username that's not already taken and that you're happy with...
  2. Send me an email with your Source Forge username so that I can add you to the list of PLearn developers.
  3. If you don't already have a ~/.ssh/identity.pub file on your computer, generate one by typing the command ssh-keygen (or ssh-keygen -t rsa). If prompted for a passphrase, simply type enter (no passphrase) that's what I use and it's probably what you want. The aim of the whole procedure is precisely to avoid having to retype passwords every time you operate with CVS... And it's very safe as long as people can't read your private ~/.ssh/identity file (if they can... well they've already hacked your account!).
  4. Once you have been added to the list of PLearn developers, log in to the sourceforge web (click on my sf.net in the tab-bar on sourceforge.net), go to your account options and click on [Edit Keys] at the bottom of the page where it says CVS/SSH Shared Authorized Keys. There, paste the content of your ~/.ssh/identity.pub file and click on Update
  5. Make sure that the environment variable CVS_RSH is always set to ssh (add export CVS_RSH=ssh to your .shrc or .bashrc file, and setenv CVS_RSH ssh to your .cshrc or .tcshrc and launch a new shell)
  6. I'll send you an email when I've added you to the list of developers. Once you receive it, you can check-out PLearn (after making sure there is no other PLearn directory in the way; rename it in PLearn.old if necessary). Simply issue the following command:
    cvs -z3 -d:ext:your_sourceforge_username@cvs.sourceforge.net:/cvsroot/plearn co PLearn
